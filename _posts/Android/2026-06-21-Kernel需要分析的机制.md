## 1. 进程与线程调度

CFS / EEVDF：普通线程调度

SCHED_FIFO、SCHED_RR：实时调度

SCHED_DEADLINE：截止时间调度

调度优先级、nice、CPU affinity

SMP 多核负载均衡

CPU idle task

调度域、调度组

Android task profile / cgroup 调度分组

uclamp：限制任务最低、最高 CPU 性能需求

Energy Aware Scheduling（EAS）：结合能耗模型选择 CPU

Vendor scheduler hooks：手机厂商定制调度策略的接口

Android 上前台应用、后台应用、System Server、音频线程等，通常会通过 cgroup、调度优先级和 uclamp 获得不同待遇。

swap / zram

内存规整 compaction

Huge Pages / THP

NUMA，虽然多数手机 SoC 上不像服务器那么突出

memfd

DMA-BUF：跨子系统共享缓冲区

DMA-BUF Heaps：Android 现代共享内存分配机制

PSI：内存、CPU、IO 压力统计

madvise、process_madvise

MGLRU：新一些 Android 设备常用的多代 LRU 回收机制

Android 的低内存处理也值得单独区分：


老版本：内核里的 Low Memory Killer
新版本：用户态 lmkd
内核负责提供 PSI、memcg 等压力和限制信息
lmkd 根据 Android 进程优先级决定杀哪个进程
所以“回收内存”主要是内核机制，“杀哪个 Android App”越来越偏向用户态策略。

## 3. 进程隔离与资源控制

cgroup v1 / cgroup v2
CPU、内存、IO、进程数等资源控制

memcg：按进程组统计和限制内存
blkio / io controller：IO 权重和限速
freezer：冻结一组进程
namespaces：
PID namespace
mount namespace
network namespace
user namespace
IPC namespace
Android 的应用沙箱不完全依赖 namespace，但 cgroup、UID、SELinux、seccomp 等组合非常关键。

Android App 沙箱大体是：


独立 UID
+ SELinux domain
+ seccomp syscall 过滤
+ 文件权限
+ Binder 权限检查
+ namespace/cgroup 等隔离
